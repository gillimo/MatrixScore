## Release Guide

Versioning policy
- Semantic versioning (MAJOR.MINOR.PATCH).
- MAJOR: breaking API or schema changes.
- MINOR: new features, backwards compatible.
- PATCH: fixes and documentation updates.

Release notes template
```
Release X.Y.Z
Date: YYYY-MM-DD

Highlights
- 

Changes
- Added:
- Changed:
- Fixed:
- Removed:

Migration notes (if needed)
- 

Checks
- Tests pass
- README updated
- Schema updated
```
